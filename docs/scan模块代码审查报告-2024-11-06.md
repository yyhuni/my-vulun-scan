## 三、潜在 Bug

### 🐛 Bug 列表

#### Bug 1: 并发竞态条件

**位置**: `scan_repository.py:437`
```python
if task_id and task_id not in scan.task_ids:
    scan.task_ids.append(task_id)
    scan.task_names.append(task_name)
    scan.save()
```

**问题**: 
- 检查和修改之间存在竞态条件
- ArrayField 的 append 操作不是原子的
- 可能导致任务重复添加或丢失

**修复建议**:
```python
from django.db.models import F
from django.contrib.postgres.aggregates import ArrayAgg

@staticmethod
@transaction.atomic
def append_task(scan_id: int, task_id: str, task_name: str) -> bool:
    # 使用数据库级别的数组操作
    updated = Scan.objects.filter(
        id=scan_id
    ).exclude(
        task_ids__contains=[task_id]  # 避免重复
    ).update(
        task_ids=F('task_ids') + [task_id],
        task_names=F('task_names') + [task_name]
    )
    return updated > 0
```

#### Bug 2: 工作空间路径冲突

**位置**: `scan_service.py:96-100`
```python
timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
unique_id = uuid.uuid4().hex[:8]
```

**问题**: 
- 时间戳精度只到秒
- 同一秒创建多个扫描仍可能冲突（虽然有 UUID）
- 但 UUID 只取 8 位，碰撞概率不是零

**修复建议**:
```python
# 使用完整 UUID 或添加毫秒
timestamp = datetime.now().strftime('%Y%m%d_%H%M%S_%f')  # 添加微秒
unique_id = uuid.uuid4().hex  # 使用完整 UUID
```

#### Bug 3: 信号处理器中的状态判断逻辑缺陷

**位置**: `status_update_handler.py:421`
```python
if scan.status == ScanTaskStatus.FAILED:
    logger.info("Scan 已处于 FAILED 状态，跳过 ABORTED 更新")
    self._trigger_workspace_cleanup(scan_id)
    return True
```

**问题**: 
- 只保护了 FAILED 状态
- SUCCESSFUL 状态也可能被 revoked 信号覆盖
- 场景：finalize_scan 将 Scan 标记为 SUCCESSFUL，但之前提交的子任务此时被撤销

**修复建议**:
```python
# 保护所有终态
FINAL_STATUSES = {
    ScanTaskStatus.SUCCESSFUL,
    ScanTaskStatus.FAILED,
    ScanTaskStatus.ABORTED
}

if scan.status in FINAL_STATUSES:
    logger.info(
        "Scan 已处于终态 %s，跳过状态更新",
        ScanTaskStatus(scan.status).label
    )
    return True
```

### 2. 敏感信息泄露

**位置**: `command_executor.py:66`

**风险**: 错误输出可能包含敏感信息（如密钥、token）

**修复建议**:

```python
def sanitize_error_output(output: str) -> str:
    """移除敏感信息"""
    patterns = [
        r'(api[_-]?key|token|password)[=:]\s*\S+',
        # ... 更多模式
    ]
    for pattern in patterns:
        output = re.sub(pattern, r'\1=***', output, flags=re.IGNORECASE)
    return output

logger.error("Error output: %s", sanitize_error_output(e.stderr.strip()))
```

### 3. 路径遍历风险

**位置**: `scan_service.py:99`
```python
workspace_path = str(Path(base_dir) / f"scan_{timestamp}_{unique_id}")
```

**风险**: 如果 `base_dir` 来自不可信源，可能导致路径遍历

**当前**: 来自环境变量，相对安全
**建议**: 添加路径验证

```python
def _validate_workspace_path(path: str, base_dir: str) -> bool:
    """验证工作空间路径在基础目录内"""
    try:
        resolved = Path(path).resolve()
        base = Path(base_dir).resolve()
        return resolved.is_relative_to(base)
    except (ValueError, OSError):
        return False
```
