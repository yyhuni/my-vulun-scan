## 5. 问题清单

### 🟠 P1 - 重要问题 (2 个)

#### 5.1.1 批量删除错误处理存在风险

**📍 位置**: `backend/internal/handlers/endpoint.go:271`

**问题描述**:
```go
// 危险的字符串切片操作
if err.Error() != "" && (err.Error()[0:2] == "请求" || err.Error()[0:2] == "端点") {
    response.BadRequestResponse(c, err.Error())
    return
}
```

**风险分析**:
- 如果错误信息长度小于 2，会导致 panic
- 依赖错误信息的前两个字符判断类型，语义脆弱

**修复建议**:
```go
// 推荐：定义业务错误类型
var (
    ErrPartialDelete = errors.New("部分ID不存在")
    ErrEmptyIDList   = errors.New("ID列表为空")
)

// 在 handler 中判断
if errors.Is(err, ErrPartialDelete) || errors.Is(err, ErrEmptyIDList) {
    response.BadRequestResponse(c, err.Error())
    return
}
```

---

#### 5.1.2 服务层忽略排序参数

**📍 位置**: 
- `backend/internal/services/endpoint.go:43`
- `backend/internal/services/endpoint.go:282`

**问题描述**:
```go
// ❌ 固定排序，忽略了 req.SortBy 和 req.SortOrder
query = query.Order("updated_at desc")
```

**修复建议**:
```go
// 白名单验证排序字段
allowedSortFields := map[string]bool{
    "url": true, "method": true, "status_code": true,
    "created_at": true, "updated_at": true,
}

sortBy := req.SortBy
if !allowedSortFields[sortBy] {
    sortBy = "updated_at"
}

sortOrder := strings.ToLower(req.SortOrder)
if sortOrder != "asc" && sortOrder != "desc" {
    sortOrder = "desc"
}

orderClause := fmt.Sprintf("%s %s", sortBy, sortOrder)
query = query.Order(orderClause)
```
