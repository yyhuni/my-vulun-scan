## 2. 严重问题（需要立即修复）

### 🔴 2.1 潜在的空指针引用风险

**严重程度：** ⭐⭐⭐⭐⭐  
**位置：** `backend/internal/services/domain.go:157-169`

#### 问题描述

在 `CreateDomains` 方法中构建结果列表时，直接使用 `existingDomainMap[detail.Name]` 可能返回零值。如果 map 中不存在该键，会导致空结构体被使用，进而创建无效的数据库记录。

#### 问题代码

```go
// 第 157-169 行
for _, detail := range req.Domains {
    domain := existingDomainMap[detail.Name]  // ❌ 如果不存在，返回零值
    
    // 如果该域名未关联到组织，添加到待创建列表
    if !existingAssocMap[domain.ID] {
        newAssocs = append(newAssocs, models.OrganizationDomain{
            OrganizationID: req.OrgID,
            DomainID:       domain.ID,  // ⚠️ domain.ID 可能为 0
        })
    }
    
    // 添加到结果列表（保持请求顺序）
    resultDomains = append(resultDomains, domain)
}
```

#### 影响范围

1. **数据完整性问题**：可能创建 `domain_id = 0` 的无效关联记录
2. **业务逻辑错误**：返回的域名列表可能包含空结构体
3. **级联问题**：后续使用这些数据的代码可能出现异常

#### 修复方案

```go
for _, detail := range req.Domains {
    // ✅ 使用标准化的域名作为 key
    normalizedName := strings.ToLower(strings.TrimSpace(detail.Name))
    
    // ✅ 检查是否存在
    domain, exists := existingDomainMap[normalizedName]
    if !exists {
        log.Error().
            Str("domain", normalizedName).
            Uint("organization_id", req.OrgID).
            Msg("Domain not found in map, this should not happen")
        continue  // 跳过不存在的域名
    }
    
    // 检查关联是否已存在
    if !existingAssocMap[domain.ID] {
        newAssocs = append(newAssocs, models.OrganizationDomain{
            OrganizationID: req.OrgID,
            DomainID:       domain.ID,
        })
    }
    
    // 添加到结果列表
    resultDomains = append(resultDomains, domain)
}
```

```go

    // 预期：跳过该域名，不创建无效关联
```

### 🔴 2.2 SQL 注入风险（字符串拼接）

**严重程度：** ⭐⭐⭐⭐  
**位置：** `backend/pkg/database/database.go:51-57`

#### 问题描述

虽然使用了 GORM，但在数据库创建逻辑中使用了字符串拼接构建 SQL 语句，存在潜在的 SQL 注入风险。

#### 问题代码

```go
// 第 51 行 - 检查数据库是否存在
checkQuery := fmt.Sprintf("SELECT COUNT(*) FROM pg_database WHERE datname = '%s'", cfg.Database.DBName)
systemDB.Raw(checkQuery).Row().Scan(&count)

// 第 57 行 - 创建数据库
createQuery := fmt.Sprintf("CREATE DATABASE %s", cfg.Database.DBName)
if err := systemDB.Exec(createQuery).Error; err != nil {
    // ...
}
```

#### 风险分析

虽然配置通常是可信的，但存在以下风险：
1. 如果配置来源被污染（环境变量注入、配置文件篡改）
2. 如果未来支持动态数据库名称
3. 数据库名称包含特殊字符（如单引号、分号）

#### 影响范围

- **低可能性，高影响**：虽然配置通常可信，但一旦被利用，可能导致：
  - 创建恶意数据库
  - 执行任意 SQL 命令
  - 数据泄露或破坏

#### 修复方案

```go
// 1. 添加数据库名称验证函数
func validateDatabaseName(name string) error {
    if name == "" {
        return fmt.Errorf("database name cannot be empty")
    }
    
    // PostgreSQL 标识符规则：字母、数字、下划线，以字母或下划线开头
    matched, err := regexp.MatchString(`^[a-zA-Z_][a-zA-Z0-9_]*$`, name)
    if err != nil {
        return err
    }
    if !matched {
        return fmt.Errorf("invalid database name: %s (only letters, numbers, underscore allowed)", name)
    }
    
    // PostgreSQL 限制：标识符最长 63 字节
    if len(name) > 63 {
        return fmt.Errorf("database name too long: %s (max 63 characters)", name)
    }
    
    // 防止使用保留关键字
    reserved := map[string]bool{
        "postgres": true, "template0": true, "template1": true,
    }
    if reserved[strings.ToLower(name)] {
        return fmt.Errorf("database name is reserved: %s", name)
    }
    
    return nil
}

// 2. 在使用前验证
func InitDB() {
    once.Do(func() {
        cfg := config.GetConfig()
        
        // ✅ 验证数据库名称
        if err := validateDatabaseName(cfg.Database.DBName); err != nil {
            initErr = err
            log.Fatal().Err(err).Msg("Invalid database name in configuration")
            return
        }
        
        // ... 其余代码
    })
}

// 3. 使用参数化查询（对于检查数据库存在）
var count int64
err := systemDB.Raw("SELECT COUNT(*) FROM pg_database WHERE datname = $1", cfg.Database.DBName).
    Row().Scan(&count)

// 注意：CREATE DATABASE 不支持参数化，但通过验证名称已经安全
```

### 🔴 2.3 配置文件缺少关键默认值

**严重程度：** ⭐⭐⭐  
**位置：** `backend/config/config.go` 和 `backend/config/config.yaml`

#### 问题描述

1. 配置文件中没有设置 `server.port` 的默认值
2. 项目规则要求后端使用 8888 端口，但代码中未强制
3. 缺少其他关键配置的默认值

#### 问题代码

```go
// config/config.go 第 73-78 行
func setDefaults() {
    // 服务器默认配置
    viper.SetDefault("server.host", "localhost")
    viper.SetDefault("server.mode", "release")
    viper.SetDefault("server.read_timeout", 15)
    viper.SetDefault("server.write_timeout", 15)
    // ❌ 缺少 server.port 的默认值
    
    // ...
}
```

#### 影响范围

1. **端口配置不明确**：如果配置文件缺少端口，程序行为不确定
2. **不符合规范**：违反了「后端应该是 8888 端口」的项目规则
3. **部署问题**：不同环境可能使用不同端口，导致混乱

#### 修复方案

```go
// config/config.go - 完善默认值设置
func setDefaults() {
    // 服务器默认配置
    viper.SetDefault("server.host", "0.0.0.0")  // ✅ 监听所有接口
    viper.SetDefault("server.port", 8888)       // ✅ 明确指定默认端口
    viper.SetDefault("server.mode", "release")
    viper.SetDefault("server.read_timeout", 15)
    viper.SetDefault("server.write_timeout", 15)
    
    // 数据库默认配置
    viper.SetDefault("database.host", getEnv("DB_HOST", "localhost"))
    viper.SetDefault("database.port", getEnvAsInt("DB_PORT", 5432))
    viper.SetDefault("database.user", getEnv("DB_USER", "postgres"))
    viper.SetDefault("database.password", getEnv("DB_PASSWORD", ""))  // ✅ 空密码作为默认
    viper.SetDefault("database.dbname", getEnv("DB_NAME", "vulun_scan"))
    viper.SetDefault("database.sslmode", getEnv("DB_SSLMODE", "disable"))
    viper.SetDefault("database.max_conns", getEnvAsInt("DB_MAX_CONNS", 25))  // ✅ 调整默认连接数
    
    // 安全默认配置
    viper.SetDefault("security.jwt_secret", "")  // ✅ 强制要求设置
    viper.SetDefault("security.jwt_expiry_hour", 24)
}

// ✅ 添加配置验证
func (c *Config) Validate() error {
    if c.Server.Port < 1 || c.Server.Port > 65535 {
        return fmt.Errorf("invalid server port: %d", c.Server.Port)
    }
    
    if c.Database.MaxConns < 1 {
        return fmt.Errorf("database max_conns must be at least 1")
    }
    
    if c.Security.JWTSecret == "" {
        return fmt.Errorf("security.jwt_secret must be set")
    }
    
    return nil
}

// 在 GetConfig 中调用验证
func GetConfig() *Config {
    if AppConfig == nil {
        if err := LoadConfig(); err != nil {
            log.Fatalf("Failed to load config: %v", err)
        }
        
        // ✅ 验证配置
        if err := AppConfig.Validate(); err != nil {
            log.Fatalf("Invalid configuration: %v", err)
        }
    }
    return AppConfig
}
```
