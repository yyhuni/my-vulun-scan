# 原因定位
- **AutoMigrate 不会修改已有外键**：即使模型里有 `constraint:OnDelete:CASCADE`，已存在的外键仍保持 `NO ACTION`。
- 你已删除并重建表，但依然是 `NO ACTION`，说明当前的迁移链路没有把 CASCADE 正确“落地”到数据库约束中（GORM 的约束生成在某些场景下并不稳定）。
