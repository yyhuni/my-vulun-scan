# 组织详情页面路由化改造方案

## 📋 方案概述

将组织详情页面从**单页 Tabs 切换**改造为**多页面路由架构**，每个资产类型（主资产、子域名、Endpoint）拥有独立的路由页面。

**核心思路**：
- Tabs 导航保留在组织详情页，但作为**路由跳转触发器**
- 点击 Tab 后跳转到对应的子路由页面
- 每个子页面独立开发，可使用表格或其他展示方式

---

## 🎯 核心目标

1. **路由独立**：每个资产类型有独立的 URL 和页面
2. **灵活展示**：不限于表格，可自由选择展示方式
3. **可分享链接**：支持直接访问特定资产类型页面
4. **浏览器导航**：支持前进/后退按钮

## 🗺️ 路由设计

### URL 结构：
```
/assets/organization/1                    → 组织详情页（显示 Tabs 导航）
/assets/organization/1/main-assets        → 主资产页面
/assets/organization/1/subdomains         → 子域名页面
/assets/organization/1/endpoints          → Endpoint 页面
```

### 导航流程：
1. 用户访问 `/assets/organization/1` → 看到组织信息和 Tabs 导航
2. 点击 "主资产" Tab → 跳转到 `/assets/organization/1/main-assets`
3. 点击 "子域名" Tab → 跳转到 `/assets/organization/1/subdomains`
4. 点击 "Endpoint" Tab → 跳转到 `/assets/organization/1/endpoints`

## 📐 实现方案

### 1. 组织详情页（`/app/assets/organization/[id]/page.tsx`）

**职责**：
- 显示组织基本信息（名称、描述）
- 渲染 Tabs 导航栏
- 点击 Tab 触发路由跳转

**核心逻辑**：
- 使用 Next.js `useRouter` 进行路由跳转
- Tabs 组件的 `onValueChange` 触发导航
- 根据当前路由高亮对应的 Tab

**不包含**：
- 不展示任何资产数据
- 不包含表格组件
- 仅作为导航入口

---

### 2. 主资产页面（`/app/assets/organization/[id]/main-assets/page.tsx`）

**职责**：
- 获取和展示主资产数据
- 渲染主资产表格或其他展示组件
- 处理主资产的增删改操作

**使用组件**：
- `MainAssetsList` - 主资产列表组件
- `MainAssetsDataTable` - 主资产表格组件
- 或其他自定义展示组件

---

### 3. 子域名页面（`/app/assets/organization/[id]/subdomains/page.tsx`）

**职责**：
- 获取和展示子域名数据

- 处理子域名的增删改操作

**使用组件**：
- `SubdomainsList` - 子域名列表组件
- `SubdomainsDataTable` - 子域名表格组件
- 或其他自定义展示组件

---

### 4. Endpoint 页面（`/app/assets/organization/[id]/endpoints/page.tsx`）

**职责**：
- 获取和展示 Endpoint 数据
- 渲染 Endpoint 表格或其他展示组件

**使用组件**：
- `EndpointsList` - Endpoint 列表组件
- `EndpointsDataTable` - Endpoint 表格组件
- 或其他自定义展示组件

## ✅ 方案优势

### 1. **路由独立性**
- 每个资产类型有独立的 URL
- 可以直接分享特定资产类型的链接
- 支持浏览器前进/后退

### 2. **展示灵活性**
- 不限于表格展示
- 可以根据资产类型特点自由设计
- 例如：Endpoint 可以用树形结构，子域名可以用图谱

### 3. **性能优化**
- 按需加载：只加载当前访问的资产类型数据
- 减少初始页面加载时间
- 降低内存占用

### 4. **开发独立性**
- 每个页面可以独立开发和测试
- 团队成员可以并行开发不同资产类型
- 减少代码冲突

### 5. **SEO 友好**
- 每个页面有独立的 URL 和元数据
- 便于搜索引擎索引

## 📝 技术要点

### 1. Tabs 导航实现
- 使用 `useRouter` 和 `usePathname` 获取当前路由
- 根据路径高亮对应的 Tab
- 点击 Tab 时使用 `router.push()` 跳转

### 2. 组织信息共享
- 可以使用 Next.js Layout 共享组织信息
- 或在每个子页面独立获取（根据性能需求选择）

### 3. 状态管理
- 每个子页面独立管理自己的状态
- 不需要全局状态管理
- 简化代码复杂度
