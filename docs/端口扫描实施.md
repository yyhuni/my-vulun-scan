## 四、核心逻辑设计

### 1. **query_subdomains_task**
```python
输入：target_id
输出：List[SubdomainDTO] (包含 id, name)

逻辑：
- 查询 Subdomain 表，条件：target_id=xxx
- 返回所有子域名（不限制扫描任务）
```

### 2. **resolve_ips_task**
```python
输入：List[subdomain_name]
输出：Dict[subdomain_name, List[ip]]

逻辑：
- 使用 dnspython 进行 A/AAAA 记录查询
- 并发解析提升速度
- 容错：解析失败的子域名跳过
```

### 3. **run_port_scanner_task**
```python
输入：ip_address, scan_type, result_dir
输出：result_file_path

工具选择：
- 快速扫描：masscan（推荐，速度快）
- 详细扫描：nmap（服务识别）

命令示例：
masscan {ip} -p1-65535 --rate=1000 -oJ {output_file}
nmap -Pn -sS -T4 -p- {ip} -oX {output_file}
```

### 4. **parse_port_results_task**
```python
输入：List[result_file_path], format
输出：List[PortDTO]

逻辑：
- 解析 JSON (masscan) 或 XML (nmap)
- 提取：ip, port, service_name, is_uncommon
- 标准化数据格式
```

### 5. **save_ports_task**
```python
输入：List[PortDTO], scan_id
输出：saved_count

逻辑：
1. 先 upsert IPAddress（去重逻辑）
2. 再 upsert Port（关联 ip_id）
3. 批量保存（batch_size=500）
4. 只保存首次发现的端口（UniqueConstraint）
```

## 八、异常处理

### 1. DNS 解析失败
```python
# 容错：跳过无法解析的子域名
logger.warning("子域名 %s 解析失败，跳过", subdomain)
continue
```

### 2. 端口扫描超时
```python
# 单个 IP 超时不影响其他
try:
    scan_port(ip, timeout=600)
except TimeoutError:
    logger.warning("IP %s 扫描超时，跳过", ip)
    return ""  # 返回空表示失败
```

### 3. 数据库冲突
```python
# UniqueConstraint 冲突 → 跳过已存在的端口
except IntegrityError:
    logger.debug("端口 %s:%d 已存在，跳过", ip, port)
```

## 九、Flow 执行流程

```python
@flow(name="port_scan", log_prints=True)
def port_scan_flow(
    scan_id: int,
    target_name: str,
    target_id: int,
    scan_workspace_dir: str,
    engine_config: str = None
) -> dict:
    """
    端口扫描流程
    
    步骤：
    1. 查询 Target 下所有子域名
    2. 解析子域名的 IP 地址
    3. 并行扫描所有 IP 的端口
    4. 解析扫描结果
    5. 批量保存到数据库
    """
    
    # Step 1: 查询子域名
    subdomains = query_subdomains_task(target_id)
    
    # Step 2: DNS 解析
    ip_mapping = resolve_ips_task([s.name for s in subdomains])
    
    # Step 3: 并行扫描端口
    futures = {}
    for subdomain, ips in ip_mapping.items():
        for ip in ips:
            future = run_port_scanner_task.submit(
                ip=ip,
                subdomain=subdomain,
                result_dir=result_dir,
                command=config['command'],
                timeout=config['timeout']
            )
            futures[f"{subdomain}_{ip}"] = future
    
    # 等待扫描完成
    results = [f.result() for f in futures.values()]
    result_files = [r for r in results if r]  # 过滤失败
    
    # Step 4: 解析结果
    ports_data = parse_port_results_task(result_files, format='json')
    
    # Step 5: 保存到数据库
    saved_count = save_ports_task(
        ports_data=ports_data,
        scan_id=scan_id,
        target_id=target_id
    )
    
    return {
        'success': True,
        'scan_id': scan_id,
        'total_ports': saved_count,
        'scanned_ips': len(ip_mapping)
    }
```

## 十一、风险点

### 1. 性能风险
- **问题**：扫描 1000 个子域名 × 65535 端口 = 海量任务
- **方案**：
  - 默认只扫描 Top 1000 端口
  - 支持通过 engine_config 配置端口范围

### 2. 工具依赖
- **问题**：masscan/nmap 未安装
- **方案**：
  - 在 Docker 镜像中预装
  - Task 启动时检测工具可用性

### 3. 网络限制
- **问题**：扫描速率过快可能被防火墙拦截
- **方案**：
  - 限制并发数（max_workers=10）
  - 配置扫描速率（--rate=1000）
