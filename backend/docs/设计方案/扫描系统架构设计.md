# XingRin 扫描系统架构设计方案

---

## 📋 一、核心需求确认

### 1.1 功能需求

#### ✅ 必须实现的功能
- **只显示新增资产**：每次扫描的 Scan 记录只显示本次新发现的资产
- **资产去重管理**：同一 Target 下的资产（Subdomain/Port/WebSite）全局唯一
- **扫描报告生成**：展示每次扫描的新增资产数量和列表

#### ❌ 不需要实现的功能
- 查询"某次扫描发现的所有资产（包括已存在的）"
- 查询"某资产在哪些扫描中出现过"
- 资产状态变更历史

### 1.2 系统特点

**核心优势**：
- 能根据 Target 查找下面的资产，且不重复
- 系统既能做扫描历史跟踪，也能作为资产管理工具

---

## 🔄 二、扫描流程设计

### 2.1 基础扫描流程

```
Target → 子域名扫描 → 端口扫描 → 站点识别（httpx）
```

### 2.2 单项扫描类型

#### 2.2.1 子域名扫描

**功能**：
- 用于最大化发现子域名资产

**逻辑流程**：
```
1. 选择 Target
   ↓
2. 运行子域名扫描工具（amass、subfinder 等）
   ↓
3. 保存新增的子域名到数据库
```

**特点**：
- 只保存首次发现的子域名
- 已存在的子域名不重复插入

---

#### 2.2.2 端口扫描

**功能**：
- 用于最大化发现端口资产

**逻辑流程**：
```
1. 选择 Target
   ↓
2. 查询 Target 下的所有 Subdomain
   ↓
3. 对每个 Subdomain 进行端口扫描
   ↓
4. 保存新增的端口到数据库
```

**特点**：
- 扫描所有已知子域名（包括历史扫描发现的）
- 只保存首次发现的端口

---

#### 2.2.3 站点扫描

**功能**：
- 用于最大化发现站点资产

**逻辑流程**：
```
1. 选择 Target
   ↓
2. 查询 Target 下的所有 Subdomain
   ↓
3. 查询每个 Subdomain 下的所有端口
   ↓
4. 拼接 Subdomain + Port，生成 URL 列表
   ↓
5. 使用 httpx 进行站点识别
   ↓
6. 保存新增的站点到数据库
```

**特点**：
- 扫描所有已知的子域名和端口组合
- 只保存首次发现的站点

---

### 2.3 组合扫描类型

#### 2.3.1 Full Scan（全量扫描）

**功能**：
- 运行所有的扫描逻辑
- 适用于定期全面检查

**逻辑流程**：
```
1. 选择 Target
   ↓
2. 【子域名扫描】
   - 运行扫描工具
   - 保存新增子域名到数据库
   ↓
3. 【端口扫描】
   - 查询 Target 下的所有 Subdomain（包括新增 + 历史）
   - 进行端口扫描
   - 保存新增端口到数据库
   ↓
4. 【站点扫描】
   - 查询 Target 下的所有 Subdomain + Port 组合
   - 使用 httpx 进行站点识别
   - 保存新增站点到数据库
```

**扫描范围**：
- ✅ 所有子域名（新增 + 历史）
- ✅ 所有端口（新增 + 历史）
- ✅ 所有站点（新增 + 历史）

**适用场景**：
- 首次扫描
- 定期全面检查
- 需要发现已有资产的新变化（如已有子域名新开放的端口）

---

#### 2.3.2 快速扫描

**功能**：
- 快速运行所有的扫描逻辑
- 适用于高频监控

**逻辑流程**：
```
1. 选择 Target
   ↓
2. 【子域名扫描】
   - 运行扫描工具
   - 保存新增子域名到数据库
   ↓
3. 【端口扫描】（仅针对新增子域名）
   - 只扫描本次新增的 Subdomain
   - 保存新增端口到数据库
   ↓
4. 【站点扫描】（仅针对新增子域名）
   - 只处理本次新增的 Subdomain + Port 组合
   - 使用 httpx 进行站点识别
   - 保存新增站点到数据库
```

**扫描范围**：
- ✅ 所有子域名（新增 + 历史）
- ⚠️ 仅新增子域名的端口
- ⚠️ 仅新增子域名的站点

**适用场景**：
- 高频监控
- 只关注新增资产
- 节省扫描时间

**注意事项**：
- 如果本次扫描没有新增子域名，后续的端口和站点扫描将跳过
- 不会发现已有子域名的新端口或新站点

## 🎯 四、核心设计原则

### 4.1 资产唯一性

**原则**：
- 同一 Target 下的资产全局唯一
- 资产只记录首次发现的扫描

**实现方式**：
- 数据库唯一约束：`unique_constraint(name, target_id)`
- 插入前先查询是否已存在，只插入新增的

### 4.2 增量数据追踪

**原则**：
- Scan 记录只关联本次新增的资产
- 通过 `scan_id` 可以查询某次扫描的新增资产

**优势**：
- 清晰展示每次扫描的价值
- 方便生成扫描报告
- 支持统计分析（每次扫描新增资产趋势）

---

## ✅ 五、总结

### 5.1 设计亮点

1. **简洁高效**：只关注新增资产，避免数据冗余
2. **灵活可扩展**：支持多种扫描类型和组合
3. **双重价值**：既是扫描工具，也是资产管理平台
