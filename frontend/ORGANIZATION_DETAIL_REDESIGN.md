# 组织详情页面重新设计

## 变更概述

将组织详情页面从使用 Tab 导航的多子页面结构简化为单一页面设计，提供更简洁直观的用户体验。

## 设计理念

### 问题
- 原设计使用 Tab 导航，但实际上只有一个"主资产"子页面
- 路由嵌套结构（`/assets/organization/[id]/assets`）过于复杂
- Tab 导航占用空间但不提供实际价值

### 解决方案
- **移除 Tab 导航**：去除不必要的导航层级
- **统一路由**：直接使用 `/assets/organization/[id]` 展示所有内容
- **统计卡片**：在页面顶部展示组织的关键指标
- **直接展示内容**：主资产列表直接显示在统计卡片下方

## 功能特性

### 1. 统计卡片
展示组织的关键指标：
- **主资产总数**：组织关联的主资产数量
- **域名总数**：已发现的域名数量
- **端点总数**：已识别的端点数量

每个卡片都包含：
- 图标标识
- 大字号数字展示
- 描述性文字说明

### 2. 主资产列表
完整的资产管理功能：
- 搜索和筛选
- 列显示/隐藏控制
- 批量删除操作
- 关联新资产
- 分页浏览

### 3. 响应式设计
- 移动端：统计卡片单列显示
- 平板：两列显示
- 桌面：三列显示

## 用户体验改进

### 优点
1. **更简洁**：减少了一层导航层级
2. **更直观**：关键信息一目了然
3. **更高效**：统计卡片快速了解组织概况
4. **更一致**：与其他单一内容页面保持一致的设计模式

### 使用场景
用户访问组织详情页面时：
1. 首先看到组织基本信息和统计数据
2. 快速了解组织规模（资产、域名、端点数量）
3. 向下滚动查看详细的资产列表
4. 可以进行资产关联、删除等操作

## 未来扩展

如果需要添加更多功能（如域名列表、端点列表），可以考虑：
1. **垂直展开式**：使用折叠面板或手风琴组件
2. **模态对话框**：点击统计卡片弹出详情
3. **单独页面**：为复杂功能创建独立页面，从统计卡片链接跳转

但在只有主资产列表的当前阶段，单页面设计是最佳选择。

## 路由变化

### 旧路由
```
/assets/organization/[id]          -> 自动跳转到 assets
/assets/organization/[id]/assets   -> 实际展示内容的页面
```

### 新路由
```
/assets/organization/[id]          -> 直接展示所有内容
```

## API 使用

页面使用以下 API：
- `useOrganization(id)` - 获取组织基本信息
- `useOrganizationAssets(id, pagination)` - 获取组织的资产列表

统计数据从组织对象的 `stats` 字段获取，如果后端未提供，前端使用占位数据。

## 样式说明

使用 shadcn/ui 组件库：
- `Card` - 统计卡片
- `Badge` - 标签展示
- `Table` - 数据表格
- `Button` - 操作按钮
- `Dialog` - 确认对话框

所有样式遵循项目的设计系统，支持亮色/暗色主题。
